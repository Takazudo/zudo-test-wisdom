## Tools by Testing Level

| Level | Tools | Install | Run Command |
|-------|-------|---------|-------------|
| [1](../testing-levels/level-1-unit-tests.mdx) | vitest | `pnpm add -D vitest` | `pnpm vitest run` |
| [1](../testing-levels/level-1-unit-tests.mdx) | jest | `pnpm add -D jest` | `pnpm jest` |
| [1](../testing-levels/level-1-unit-tests.mdx) | vitest + Miniflare (backend/Workers) | `pnpm add -D vitest miniflare` | `pnpm vitest run` |
| [1](../testing-levels/level-1-unit-tests.mdx) | vitest + @cloudflare/vitest-pool-workers (in-worker/Workers) | `pnpm add -D vitest @cloudflare/vitest-pool-workers` | `pnpm vitest run` |
| [2](../testing-levels/level-2-dom-tests.mdx) | vitest + jsdom | `pnpm add -D vitest jsdom` | `pnpm vitest run` |
| [2](../testing-levels/level-2-dom-tests.mdx) | vitest + happy-dom | `pnpm add -D vitest happy-dom` | `pnpm vitest run` |
| [2](../testing-levels/level-2-dom-tests.mdx) | @testing-library/react | `pnpm add -D @testing-library/react` | (used in tests) |
| [2](../testing-levels/level-2-dom-tests.mdx) | vitest browser mode (@vitest/browser + Playwright) | `pnpm add -D vitest @vitest/browser playwright` | `pnpm vitest run --project browser` |
| [3](../testing-levels/level-3-build-output.mdx) | vitest (reading built files) | `pnpm add -D vitest` | `pnpm build && pnpm vitest run --project build` |
| [3](../testing-levels/level-3-build-output.mdx) | cheerio + fs/path | `pnpm add -D cheerio` | (used in tests) |
| [3](../testing-levels/level-3-build-output.mdx) / [4](../testing-levels/level-4-e2e-browser.mdx) | link check + html-validate ([site-integrity gates](../real-world-patterns/site-integrity-checks.mdx)) | `pnpm add -D html-validate` | `pnpm build && pnpm check:html && pnpm check:links` |
| [4](../testing-levels/level-4-e2e-browser.mdx) | Playwright | `pnpm add -D @playwright/test` then `npx playwright install --with-deps` | `npx playwright test` |
| [4](../testing-levels/level-4-e2e-browser.mdx) | headless-browser | (Claude Code skill) | `/headless-browser` |
| [5](../testing-levels/level-5-visual-verification.mdx) | verify-ui | (Claude Code skill) | `/verify-ui` |
| [5](../testing-levels/level-5-visual-verification.mdx) | headless-browser | (Claude Code skill) | `/headless-browser` |
| [5](../testing-levels/level-5-visual-verification.mdx) | Playwright `toHaveScreenshot()` ([visual regression baselines](../real-world-patterns/visual-regression-baselines.mdx)) | `pnpm add -D @playwright/test` | `npx playwright test --update-snapshots` (baseline) / `npx playwright test` (verify) |
| [6](../testing-levels/level-6-ai-based-verification.mdx) | verify-ui-ai (final resort, **not for CI**) | (Claude Code skill) | `/verify-ui-ai` |

Bare `vitest` starts watch mode and never exits -- use `vitest run` for one-shot/CI/agent runs. The `@playwright/test` package alone does not include browser binaries; `playwright install --with-deps` downloads them plus their OS-level dependencies.

## Vitest Configuration Quick Reference

### Minimal unit test setup

```typescript
// vitest.config.ts
import { defineConfig } from "vitest/config";

export default defineConfig({
  test: {
    include: ["src/**/*.test.ts"],
  },
});
```

### Component test setup (jsdom)

```typescript
// vitest.config.ts
import { defineConfig } from "vitest/config";

export default defineConfig({
  test: {
    environment: "jsdom",
    include: ["src/**/*.test.tsx"],
    setupFiles: ["./test-setup.ts"],
  },
});
```

### Projects setup (multiple test types)

```typescript
// vitest.config.ts
import { defineConfig } from "vitest/config";

export default defineConfig({
  test: {
    projects: [
      {
        test: { name: "unit", include: ["src/**/*.test.ts"], environment: "node" },
      },
      {
        test: {
          name: "component",
          include: ["src/**/*.test.tsx"],
          environment: "jsdom",
        },
      },
      {
        test: {
          name: "build",
          include: ["tests/build/**/*.test.ts"],
          environment: "node",
        },
      },
    ],
  },
});
```

## Playwright Configuration Quick Reference

### Basic setup

```typescript
// playwright.config.ts
import { defineConfig } from "@playwright/test";

export default defineConfig({
  testDir: "./e2e",
  use: {
    baseURL: "http://localhost:3000",
  },
  webServer: {
    command: "pnpm dev",
    port: 3000,
    reuseExistingServer: !process.env.CI,
  },
});
```

### WebKit-only (for Tauri)

```typescript
// playwright.config.ts
import { defineConfig, devices } from "@playwright/test";

export default defineConfig({
  projects: [
    { name: "webkit", use: { ...devices["Desktop Safari"] } },
  ],
});
```

### Production build testing

```typescript
// playwright.config.ts
import { defineConfig } from "@playwright/test";

export default defineConfig({
  webServer: {
    command: "pnpm build && pnpm preview",
    port: 4173,
    reuseExistingServer: !process.env.CI,
  },
});
```
